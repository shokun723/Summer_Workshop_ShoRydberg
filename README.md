# Summer_Workshop_Data_Driven_Decision_Making

Data Driven Decision Making Summer Workshop 2026

This repository contains the notebooks and setup materials for a one-week, hands-on machine learning workshop for community college students. The workshop is designed as a lab-first introduction: students run prepared notebooks, change important choices, interpret results, and connect machine learning outputs to devices they fabricate and image during the week.

## Teaching Order

The main class notebooks are in `0_Class_Notebooks`:

1. `01_Know_Data_Numerical.ipynb` - numerical data, missing values, plotting, regression, and prediction limits.
2. `02_Know_Data_Images.ipynb` - images as data, pixels, resizing, color modes, scaling, compression, and augmentation concepts.
3. `03_Image_Augmentation_Workbench.ipynb` - creating balanced image datasets and localization-ready images.
4. `04_Image_Classification_Grayscale.ipynb` - supervised image classification using grayscale device images.
5. `05_Image_Classification_RGB.ipynb` - supervised image classification using RGB device images.
6. `06_Defect_Localization_Clustering.ipynb` - clustering and anomaly-style workflows for defect localization.
7. `07_Student_ML_Experiment_Log.ipynb` - student mini-project planning, result tracking, and final reflection.

Student reference handout:

- `PYTHON_ML_CHEAT_SHEET.md` - beginner-friendly Python and machine learning command dictionary grouped by common workshop tasks.
- `Deliverables/PYTHON_ML_CHEAT_SHEET.pdf` - printable PDF version of the same handout for students.

Instructor/student deliverables:

- `Deliverables/Data_Driven_Decision_Making_Notebook_Visual_Aids.pptx` - visual aid deck aligned with the notebooks.
- `Deliverables/notebook_visual_aid_slide_images/contact_sheet.png` - quick preview sheet for the visual aid deck.

## Week Arc

- Day 1: What is data? What is a model? Use numerical data and regression to introduce prediction and trust.
- Day 2: Images are data. Explore pixels, color, resizing, compression, and the choices that shape image models.
- Day 3: Classification. Train models, compare results, inspect mistakes, and discuss supervised learning.
- Day 4: Clustering and defect localization. Use unlabeled structure and image differences to reason about defects.
- Day 5: Student mini-project. Each group chooses a question, runs an experiment, and makes a recommendation.

The central theme is: from device to data to decision.
